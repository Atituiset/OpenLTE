## 2. 实验 1：离线扫描预录的 IQ 文件（最简单）

仓库自带一个测试 IQ：`Test captures/LTE_test_file_int8.bin`。

```bash
cd /root/Projects/OpenLTE/build
./LTE_fdd_dl_file_scan/LTE_fdd_dl_file_scan \
    "/root/Projects/OpenLTE/Test captures/LTE_test_file_int8.bin"
```

你应该能看到：
- 程序在做 PSS/SSS 同步，输出"找到 PCI = N"
- 解出 MIB（DL_BW、PHICH、SFN）
- 解出 SIB1（PLMN、TAC、cellID …）
- 一段时间后退出

> **实验目的**：直观感受 PHY → MIB → SIB1 的解码过程。同时在你的脑子里把 03 篇的 MIB 24 bit 和真实数据对上。

如果你想用 Octave/Matlab 做一遍同样的事，README §11 给了三行命令：
```octave
fid = fopen("/path/to/LTE_test_file_int8.bin", "r");
iq_vec = fread(fid, inf, "int8");
iq_split_vec = reshape(iq_vec, 2, []);
lte_fdd_dl_receive(iq_split_vec(1,:) + i*iq_split_vec(2,:));
```
配合 `octave/lte_fdd_dl_*.m`，能看到每个 OFDM 符号、每个 RE 的可视化。

## 3. 实验 2：跑起来 LTE_fdd_enodeb，连上 telnet 控制台

不需要 SDR——程序会启动控制端口，但因为没 SDR 不会真发射。

终端 A：
```bash
./LTE_fdd_enodeb/LTE_fdd_enodeb
```

应该看到：
```
*** LTE FDD ENB ***
Please connect to control port 30000
```

终端 B：
```bash
telnet 127.0.0.1 30000
```

输入 `help` 列出所有命令。常用：
```
write band 20
write bandwidth 5
write dl_earfcn 6300
write mcc 214
write mnc 12
write n_ant 1
write rx_gain 30
write tx_gain 86
read_param all       # 查所有参数
add_user imsi=001010000000001 imei=000000000000001 k=00112233445566778899AABBCCDDEEFF
print_users
```

终端 C（可选）：
```bash
telnet 127.0.0.1 30001    # 调试日志，会持续吐协议事件
```

> **实验目的**：把 §05 的状态机入口熟悉一下。`add_user` 后日志里你会看到 HSS 把 IMSI 收下；想 attach 还得真有手机。

## 4. 实验 3：用 Wireshark 抓基站内部消息

在 `LTE_fdd_enodeb` 跑起来之后，按 README §"Wireshark configuration"配置：

```
Wireshark → Edit → Preferences → Protocols → DLT_USER → Edit
点 + → DLT = User 0, Payload protocol = mac-lte-framed
```

OpenLTE 会在本地 UDP 上输出 `mac-lte-framed` 格式的 PDU，Wireshark 监听 lo 接口就能看到。

让一台 UE（真手机或 srsUE）附着上来，你就能看到：
- MAC PDU
- 内嵌 RLC PDU
- 内嵌 PDCP PDU
- 解出来的 RRC PDU 和 NAS PDU（**这就是你 03/04 篇看的东西**）

> **实验目的**：把 03/04 篇里讲的字段在真实抓包里——眼睛——看一遍。
> Wireshark 的 LTE-RRC / NAS-EPS 解析器**自带 ASN.1 PER 解码**——所以你看到的字段会和 OpenLTE 解出来的完全一样。

## 5. 实验 4：加 printf 跟读 PER 编码

学习 PER 最有效的练习。在 `liblte/src/liblte_common.cc:62`（`liblte_value_2_bits`）的开头加：

```c
fprintf(stderr, "  pack %u bits = 0x%x\n", N_bits, value);
```

重新编译，跑实验 1 的离线扫描。日志里你会看到 PHY 解出 MIB 之后，调用 `liblte_rrc_pack_master_information_block_msg` 重打包做回环测试时，**24 比特一比特一比特地被吐出来**——3 + 3 + 8 + 10。

把这种 print 加到 `liblte_bits_2_value` 上，效果更直观——你会看到每一条 SIB1 的字段被一个一个解出来。

> **实验目的**：在脑子里把 03 篇的"PER 比特拼接"变成肌肉记忆。

## 6. 实验 5：手写一条 NAS 消息

挑战 04 篇说过的 Authentication Request（35 字节），用 Python 自己造一条，再用 `liblte_mme_unpack_authentication_request_msg` 解：

```python
import struct

# Plain NAS + EMM
sht_pd = (0 << 4) | 0x7
msg_type = 0x52         # AUTHENTICATION_REQUEST
ksi_spare = 0x00        # NAS KSI=0, spare half=0

rand = bytes(range(0x10, 0x20))   # 16 byte
autn = bytes(range(0x20, 0x30))   # 16 byte

frame = bytes([sht_pd, msg_type, ksi_spare]) + rand + autn
print(frame.hex())     # 看看自己造出来的字节流
print(len(frame))      # 应该是 35
```

把 `frame` 喂给 OpenLTE 的解码函数（写一个最小 C 测试程序），观察解出来的 `auth_req->rand` 和你输入的是否一致。

> **实验目的**：把 04 篇的"字节对齐 TLV/V"亲手做一遍——写一次比读三遍源码顶用。

## 7. 实验 6：跟一遍 Attach 全流程的状态切换

在 `LTE_fdd_enb_rb.cc` 的 `set_mme_state` 和 `set_rrc_state` 函数里加：

```cpp
fprintf(stderr, "[STATE] mme: %s -> %s\n",
        LTE_fdd_enb_mme_state_text[get_mme_state()],
        LTE_fdd_enb_mme_state_text[state]);
```

跑起 eNodeB，让真 UE 或 srsUE 附着，**你会看到 §05 那张时序图按顺序在日志里跑出来**：
```
[STATE] mme: IDLE -> AUTHENTICATE
[STATE] mme: AUTHENTICATE -> ENABLE_SECURITY
[STATE] mme: ENABLE_SECURITY -> RRC_SECURITY
[STATE] mme: RRC_SECURITY -> ESM_INFO_TRANSFER
[STATE] mme: ESM_INFO_TRANSFER -> ATTACH_ACCEPT
[STATE] mme: ATTACH_ACCEPT -> SETUP_DRB
[STATE] mme: SETUP_DRB -> ATTACHED
```

> **实验目的**：把 05 篇的状态机从概念变成你能在日志里"看到"的东西。

## 8. 实验 7：观察一次 Milenage 的真实运算

在 `liblte_security.cc` 的 `liblte_security_milenage_f2345` 里加 hex-dump：

```cpp
// 函数开头
fprintf(stderr, "[Milenage f2345] K=");  for (int i=0;i<16;i++) fprintf(stderr,"%02x", k[i]);   fprintf(stderr, "\n");
fprintf(stderr, "[Milenage f2345] OPc=");for (int i=0;i<16;i++) fprintf(stderr,"%02x", op_c[i]);fprintf(stderr, "\n");
fprintf(stderr, "[Milenage f2345] RAND=");for (int i=0;i<16;i++) fprintf(stderr,"%02x", rand[i]);fprintf(stderr, "\n");
// 函数末尾
fprintf(stderr, "[Milenage f2345] RES="); for (int i=0;i<8;i++)  fprintf(stderr,"%02x", res[i]); fprintf(stderr, "\n");
fprintf(stderr, "[Milenage f2345] CK=");  for (int i=0;i<16;i++) fprintf(stderr,"%02x", ck[i]);  fprintf(stderr, "\n");
fprintf(stderr, "[Milenage f2345] IK=");  for (int i=0;i<16;i++) fprintf(stderr,"%02x", ik[i]);  fprintf(stderr, "\n");
```

让任何 UE 触发一次鉴权，观察输出。

如果你愿意写一段 Python（用 `cryptography` 或 `pycryptodome` 做 AES-128），把同样的 K/OP/RAND 喂进去算 RES，**你应该得到完全一致的结果**。这一刻你会真切地"算出"过 LTE 的鉴权——非常有满足感。

> **实验目的**：让 06 篇的 Milenage 从纸上的 f1/f2/f3/f4/f5 变成"我亲手算出过"。

## 9. 进阶实验（需要 SDR）

这些需要 USRP / BladeRF：

### 9.1 让真手机附着上 OpenLTE
1. 拿一张可写 USIM（推荐 sysmoUSIM-SJS1）。
2. 按 README §7 用 pySim/sysmo-usim-tool 烧入 (IMSI, K, OP=11111111... 或 OpenLTE 默认值)。
3. 启动 `LTE_fdd_enodeb`，配 EARFCN 6300（Band 20，800 MHz）。
4. `add_user imsi=... imei=... k=...` 注册。
5. 手机插卡开机 → 等附着。
6. 抓 mac-lte-framed 看完整的 RRC + NAS 流程。

> **法律警告**：仅在屏蔽箱、屏蔽线缆或合法实验频段操作。**绝不要**用商用频段对外发射。

### 9.2 NAT + IP 上网
README 仓库根有 `enodeb_nat_script.sh`。附着完成后，OpenLTE 把 UE 的 IP 包通过 TUN 交给 Linux 内核，再由这个脚本做 NAT，把流量送出局域网。这是把"我建了一个能发数据的 LTE 私有网"实现完整的最后一步。

### 9.3 攻防研究
README 提到一系列分支（`attach_reject`、`dos_tau_reject` 等）实现了已知 LTE NAS/RRC 漏洞利用。结合 04/05 篇看，这些分支只是在状态机里替换响应消息——非常容易看懂。

> **使用须知**：仅用于**授权过的安全研究/CTF/教学**。
