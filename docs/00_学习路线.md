# 00 学习路线

> 这是为"刚接触无线通信、但是熟悉软件工程"的读者写的 OpenLTE 学习路线。
> 目标：通过 OpenLTE 这个开源 LTE 协议栈实现，把 LTE 的**信令层（RRC/NAS）** 看懂，能够在源码里逐行跟读真实的 3GPP 协议消息编解码与状态机。物理层（PHY）只浅讲，不去啃 IQ 信号、Turbo 译码、信道估计这些细节。

---

## 0. 你将要学到什么

- LTE 网络的角色分工：UE / eNodeB / EPC（MME / S-GW / P-GW / HSS）
- LTE 协议栈分层：PHY → MAC → RLC → PDCP → RRC + 控制面顶层 NAS
- **ASN.1 PER（uPER）编码**：3GPP 36.331 RRC 消息怎么压成比特流的（类比"超紧凑版的 protobuf"）
- **NAS TLV/IEI 编码**：3GPP 24.301 NAS 消息怎么压成字节流的（类比"显式带类型标签的二进制结构体"）
- **eNodeB 状态机**：RRC 连接建立、附着（Attach）、鉴权、安全模式、默认承载建立的完整流程
- **LTE 安全**：Milenage（USIM 端）+ KASME 派生 + EEA 加密 / EIA 完整性

## 1. 推荐阅读顺序

把本目录下文档按数字顺序读，先建总览再钻细节：

```
00 学习路线（本文）              ← 你正在这里
01 项目总览                       仓库结构、五大可执行程序、三个内部库
02 LTE 协议栈速览                 谁是谁、谁调用谁、信令面 vs 用户面
03 RRC 与 ASN.1 PER（重头）       从 value_2_bits 走到 MIB / SIB1 / RRCConnectionRequest
04 NAS 与 TLV（重头）             从 PD/SHT 头字节走到 Attach Request / Authentication Request
05 eNodeB 状态机与调用流          RRC + EMM 两个状态机、消息队列、Singleton 多线程
06 安全与鉴权                     Milenage f1-f5、KASME、EEA/EIA、USIM 协作
07 物理层速览（轻讲）             帧结构、各信道是干什么的；不展开信号处理
08 实验与练习                     用测试 IQ 文件扫描、telnet 控制台、Wireshark 抓包
```

第 03、04 篇是最该花时间的——它们对应你最想搞清楚的"协议消息怎么变成字节流"。

## 2. 阅读时的几个核心心智模型

这些是后面所有文档反复用到的类比。先在脑子里建好坐标系，看代码就快。

| 你熟悉的概念 | LTE 中对应的东西 | 文档里展开的位置 |
|---|---|---|
| TCP 状态机 | RRC 状态机（IDLE / SRB1_SETUP / RRC_CONNECTED ...） | 05 |
| TCP 三次握手 | RRC Connection Request / Setup / Setup Complete | 05 |
| TLS 握手中的 PSK | USIM 中的 K 与 OP，KASME 由 K + RAND/AUTN 派生而来 | 06 |
| HTTP 的 TLS 升级 | NAS 的 Security Mode Command，把后续 NAS 消息切到加密+完整性保护 | 04, 06 |
| protobuf wire format | NAS TLV/IEI 编码（字节对齐、显式 tag） | 04 |
| 紧凑二进制序列化（如 MessagePack 但更狠） | ASN.1 uPER（比特对齐、几乎没有冗余） | 03 |
| OAuth 重定向 + 颁发 token | 附着（Attach）= 鉴权 + 派生密钥 + 分配 GUTI + 拉默认 PDN 承载 | 04, 05 |
| Linux 网卡的 sk_buff | `LIBLTE_BIT_MSG_STRUCT` / `LIBLTE_BYTE_MSG_STRUCT`（消息缓冲区） | 03 |

## 3. 学习里程碑

读完每篇后，你应该能做下面这件事——做不到就回头复读。

- **读完 01**：能在不看源码的情况下，说出 `liblte/`、`libtools/`、`cmn_hdr/`、`LTE_fdd_enodeb/` 各自的职责，以及五个可执行程序里你应该最先关心哪个。
- **读完 02**：能用一句话说出每一层（PHY/MAC/RLC/PDCP/RRC/NAS）"接收"什么、"产出"什么，以及它的"上层"和"下层"是谁。
- **读完 03**：能手工把一个 24 bit 的 MIB 在纸上拆出 dl_bw / phich_config / sfn_div_4 三个字段，并指出 `liblte_rrc.cc` 里对应的代码行。
- **读完 04**：能在不看代码的情况下，说出一条 NAS 消息开头第一个字节里"高 4 位是什么、低 4 位是什么"，并解释为什么 NAS 是字节对齐而 RRC 是比特对齐。
- **读完 05**：能画出 UE 从开机到完成附着、拿到默认承载的完整时序图，**并标出 OpenLTE 源码里每一步对应的状态切换发生在哪个文件**。
- **读完 06**：能解释为什么 USIM 里只有 K 和 OP，但 eNodeB 那侧能算出和 USIM 一样的 RES/CK/IK——而手机被偷了，运营商也不需要换 K。
- **读完 07**：能说出 PSS/SSS/PBCH/PDCCH/PDSCH/PUCCH/PUSCH 各是干什么的（一句话即可），不需要懂 OFDM 的内部数学。
- **读完 08**：能起一个 LTE_fdd_enodeb，挂上 telnet 控制台，并用 Wireshark 看到至少一条 mac-lte-framed 帧。

## 4. 一些前置常识（不必精通）

- **比特、字节顺序**：LTE 大量使用 MSB-first 比特序、网络字节序的多字节整数。
- **3GPP 规范**：本系列文档会大量引用 36.331（RRC）、24.301（NAS）、33.401（安全）、35.206（Milenage）。**不需要全部读完**，遇到具体字段时按章节号去查一下即可。
- **C/C++ 阅读能力**：能跟着函数指针、结构体、状态机 switch-case 跳转。
- **基本网络协议知识**：TCP/IP 三次握手、TLS 握手大致流程。LTE 的握手在概念上极其类似。
