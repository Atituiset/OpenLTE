# 01 项目总览

> 你拿到的这个仓库是 [openlte](https://sourceforge.net/p/openlte) 在 GitHub 上的克隆 + 一些 fork 改动。它实现了一个**几乎完整、单机可跑**的 LTE FDD eNodeB（含简化版 EPC），能用 USRP / BladeRF 之类的 SDR 真的对外发射 LTE 信号，让一台已经写好了 USIM 卡的手机附着上来。

如果你已经做过 SDN/网络协议方面的开源项目，可以先把它当作"一个跑得起来的 LTE 实验平台 + 一套可独立使用的协议库"来理解。

---

## 2. 五个可执行程序，先看哪一个？

| 目录 | 作用 | 适合最先看的程度 |
|---|---|---|
| `LTE_fdd_enodeb` | **完整 eNodeB**（PHY+MAC+RLC+PDCP+RRC+简化 MME/HSS+IP gateway） | ★★★★★ 你绝大部分时间都泡在这里 |
| `LTE_fdd_dl_file_scan` | 把已录的 IQ 文件里的 LTE 下行信号解出来 | ★★★ 想看 PHY 上行流程时再看 |
| `LTE_fdd_dl_scan` | 实时频谱扫小区 | ★ 仅工具 |
| `LTE_fdd_dl_file_gen` | 生成 LTE 下行 IQ 测试文件 | ★ 仅工具 |
| `LTE_file_recorder` | 录原始 IQ | ★ 仅工具 |

> 推荐：99% 的精力放在 `LTE_fdd_enodeb/` 上，其他四个按需翻。

## 3. 三个支撑库的角色

### 3.1 `cmn_hdr/` —— 全项目通用头

只有几个头文件：basic typedefs（`uint8/uint16/uint32`）、`__attribute__((packed))` 之类的宏。**不会被你大量阅读**，知道存在即可。

### 3.2 `libtools/` —— 非协议工具

socket 包装、bit-shift、CRC、FFT 之类**不属于 LTE 协议本身**的杂活。物理层会用，但你看 RRC/NAS 一般用不到。

### 3.3 `liblte/` —— **协议库（重点）**

这是整个仓库的精华。它把 LTE 协议栈中"纯协议、不依赖具体硬件、不依赖具体调度策略"的部分抽出来做成一个可独立使用的 C++ 库。`hdr/` 里全是头文件，`src/` 是实现。

```
liblte/
├── hdr/
│   ├── liblte_common.h      ★ 基础类型 LIBLTE_BIT_MSG_STRUCT / LIBLTE_BYTE_MSG_STRUCT
│   ├── liblte_rrc.h         ★ RRC 消息定义（36.331）+ pack/unpack 接口
│   ├── liblte_mme.h         ★ NAS（EMM/ESM）消息定义（24.301）+ pack/unpack 接口
│   ├── liblte_security.h    ★ Milenage / EEA / EIA / KDF
│   ├── liblte_phy.h           物理层处理（OFDM、Turbo、scrambling 等）
│   ├── liblte_mac.h           MAC PDU
│   ├── liblte_rlc.h           RLC PDU（TM/UM/AM 三种模式）
│   └── liblte_interface.h     公共枚举（频段、带宽、CFI ...）
└── src/
    ├── liblte_common.cc       value_2_bits / bits_2_value：所有 PER 的基石
    ├── liblte_rrc.cc      ★★ 1.3 万行，36.331 几乎所有消息的 PER 编解码
    ├── liblte_mme.cc      ★★ 1.2 万行，24.301 几乎所有消息的 TLV 编解码
    ├── liblte_security.cc     Milenage、KASME、EEA/EIA 实现
    ├── liblte_phy.cc          1.4 万行，物理层（你只需要扫一眼）
    ├── liblte_mac.cc
    └── liblte_rlc.cc
```

记住四个文件就好：
- **`liblte_common.{h,cc}`** —— 基础数据结构 & 比特读写
- **`liblte_rrc.{h,cc}`** —— RRC（PER 编解码的所有秘密）
- **`liblte_mme.{h,cc}`** —— NAS（TLV/IEI 编码的所有秘密）
- **`liblte_security.{h,cc}`** —— 鉴权与密钥派生

后面的 03、04、06 三篇文档就是把这四个文件展开来讲。

### 4.1 一个反直觉但很重要的事实

OpenLTE **没有实现 S1AP 协议**。一个真实部署里，eNodeB 与 EPC 中的 MME 是两个独立物理实体，中间通过 **S1-MME 接口（S1AP）** 相连。OpenLTE 把 MME、HSS 都当成一个**子系统嵌进 eNodeB 进程**，靠**进程内消息队列**通信，省去了所有 S1AP 编解码逻辑。

这件事的好处：你看协议时少了一层壳，能直接从 RRC 跳到 NAS。
这件事的代价：本仓库不能直接帮你学习 S1AP；要学 S1AP 得去 srsRAN / Open5GS。

### 4.2 单进程多线程 + Singleton

每个子系统都是 **Singleton + pthread_mutex 保护内部状态**：

```cpp
// 取自 LTE_fdd_enb_main.cc:63
LTE_fdd_enb_interface *interface = LTE_fdd_enb_interface::get_instance();
LTE_fdd_enb_cnfg_db   *cnfg_db   = LTE_fdd_enb_cnfg_db::get_instance();
LTE_fdd_enb_hss       *hss       = LTE_fdd_enb_hss::get_instance();
// ... 各子系统全部 get_instance()
```

每个子系统跑在自己的线程里，通过 `LTE_fdd_enb_msgq` 投递事件给"邻居"。这种设计你可以类比成：**每个子系统是一个 actor，线程间通信是显式消息队列**。

## 6. 入门路径建议

按下面顺序在源码里"散步"一遍，建立物理直觉：

1. `LTE_fdd_enodeb/src/LTE_fdd_enb_main.cc` —— 看程序入口怎么把所有子系统拉起来。
2. `liblte/hdr/liblte_common.h` —— 看最基础的数据结构。
3. `liblte/src/liblte_common.cc` —— 看 `liblte_value_2_bits` / `liblte_bits_2_value`，**整个 RRC 编解码的"加法器"**。
4. `liblte/src/liblte_rrc.cc:12966` —— 看 MIB 怎么打包（最小 PER 例子，24 bit）。
5. `liblte/src/liblte_mme.cc:6753` —— 看 Authentication Request 怎么打包（最小 NAS 例子）。
6. `LTE_fdd_enodeb/hdr/LTE_fdd_enb_rb.h` —— 看 RRC、EMM 状态机的所有状态定义。
7. `LTE_fdd_enodeb/src/LTE_fdd_enb_rrc.cc` / `LTE_fdd_enb_mme.cc` —— 看状态机怎么跑。
